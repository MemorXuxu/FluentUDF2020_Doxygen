33/*
34 * -----------------------------------------------------------------
35 * Function : BandGBTRF
36 * -----------------------------------------------------------------
37 * Usage : ier = BandGBTRF(A, p);
38 * if (ier != 0) ... A is singular
39 * -----------------------------------------------------------------
40 * BandGBTRF performs the LU factorization of the N by N band
41 * matrix A. This is done using standard Gaussian elimination
42 * with partial pivoting.
43 *
44 * A successful LU factorization leaves the "matrix" A and the
45 * pivot array p with the following information:
46 *
47 * (1) p[k] contains the row number of the pivot element chosen
48 * at the beginning of elimination step k, k=0, 1, ..., N-1.
49 *
50 * (2) If the unique LU factorization of A is given by PA = LU,
51 * where P is a permutation matrix, L is a lower triangular
52 * matrix with all 1's on the diagonal, and U is an upper
53 * triangular matrix, then the upper triangular part of A
54 * (including its diagonal) contains U and the strictly lower
55 * triangular part of A contains the multipliers, I-L.
56 *
57 * BandGBTRF returns 0 if successful. Otherwise it encountered
58 * a zero diagonal element during the factorization. In this case
59 * it returns the column index (numbered from one) at which
60 * it encountered the zero.
61 *
62 * Important Note: A must be allocated to accommodate the increase
63 * in upper bandwidth that occurs during factorization. If
64 * mathematically, A is a band matrix with upper bandwidth mu and
65 * lower bandwidth ml, then the upper triangular factor U can
66 * have upper bandwidth as big as smu = MIN(n-1,mu+ml). The lower
67 * triangular factor L has lower bandwidth ml. Allocate A with
68 * call A = BandAllocMat(N,mu,ml,smu), where mu, ml, and smu are
69 * as defined above. The user does not have to zero the "extra"
70 * storage allocated for the purpose of factorization. This will
71 * handled by the BandGBTRF routine.
72 *
73 * BandGBTRF is only a wrapper around bandGBTRF. All work is done
74 * in bandGBTRF works directly on the data in the DlsMat A (i.e.,
75 * the field cols).
76 * -----------------------------------------------------------------
77 */
78
79SUNDIALS_EXPORT long int BandGBTRF(DlsMat A, long int *p);
80SUNDIALS_EXPORT long int bandGBTRF(realtype **a, long int n, long int mu, long int ml,
81 long int smu, long int *p);

86 * -----------------------------------------------------------------
87 * Usage : BandGBTRS(A, p, b);
88 * -----------------------------------------------------------------
89 * BandGBTRS solves the N-dimensional system A x = b using
90 * the LU factorization in A and the pivot information in p
91 * computed in BandGBTRF. The solution x is returned in b. This
92 * routine cannot fail if the corresponding call to BandGBTRF
93 * did not fail.
94 *
95 * BandGBTRS is only a wrapper around bandGBTRS which does all the
96 * work directly on the data in the DlsMat A (i.e., the field cols).
97 * -----------------------------------------------------------------
98 */
