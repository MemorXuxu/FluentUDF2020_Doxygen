93/*
94 * A common approach, and an approach aligned with the theory behind
95 * heaps, is to store the heap in an array. Any binary tree can be
96 * stored in an array, but because a heap is always an almost complete
97 * binary tree, it can be stored compactly. No space is required for
98 * pointers; instead, the Parent and children of each node can be
99 * found by simple arithmetic on array indices. Details depend on the
100 * root position (which in turn may depend on constraints of a
101 * programming language used for implementation). If the tree root
102 * Item has index 0 (n tree elements are a[0] .. a[n-1]), then for
103 * each index i, element a[i] has children a[2i+1] and a[2i+2], and
104 * the Parent a[floor((i-1)/2)], as shown in the figure. If the root
105 * is a[1] (tree elements are a[1] .. a[n]), then for each index i,
106 * element a[i] has children a[2i] and a[2i+1], and the Parent
107 * a[floor(i/2)]. This is a simple example of an implicit data structure.
108 *
109 * http://en.wikipedia.org/wiki/Binary_heap#Heap_operations
110 */
