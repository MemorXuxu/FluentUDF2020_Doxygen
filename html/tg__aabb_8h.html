Detailed Description

Header file for TG_AABB functions.

aabb.h

Provides functionality for AxialyAligned bounding box (AABB), based on OBB work by Gottschalk and Manocha (SIGGRAPH '96), and others.

Oriented boxes provide a 'tightest-fitting' bounding-volume, which typically works well for complex non-planar geometries, at some additional expense related to intersection tests, etc. The implementation expects a 'TG_Container' struct, which provides, at a minimum:

  • size: The size of the data
  • nComponents: The number of vector components
  • nPoints: Function that returns the number of points represented by a requested element
  • getPoint: Function that returns the n-th point for a requested element
  • side: Function that returns the side of a requested element, when provided a plane which passes through a point, and a supplied normal (may not be normalized).
    • Returns 0 if on the plane
    • Returns +1 if on the same side as the normal
    • Returns -1 if on the opposite side of the normal

Using this container as a base, additional data may be held in a derived structure for purposes of maintaining state. The idea here is that since your data set is probably large, you do not want to copy it and you probably don't want to store your data in a different type just so you can do geometric searches on it. Instead, you can use the Facade pattern to create the interface described above for your data, and then use the facade as the container for this implementation to perform geometric searches.
