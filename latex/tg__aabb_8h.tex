\doxysubsection{Detailed Description}
Header file for TG\+\_\+\+AABB functions. 

aabb.\+h

Provides functionality for Axialy\+Aligned bounding box (AABB), based on OBB work by Gottschalk and Manocha (SIGGRAPH \textquotesingle{}96), and others.

Oriented boxes provide a \textquotesingle{}tightest-\/fitting\textquotesingle{} bounding-\/volume, which typically works well for complex non-\/planar geometries, at some additional expense related to intersection tests, etc. The implementation expects a \textquotesingle{}TG\+\_\+\+Container\textquotesingle{} struct, which provides, at a minimum\+:


\begin{DoxyItemize}
\item size\+: The size of the data
\item n\+Components\+: The number of vector components
\item n\+Points\+: Function that returns the number of points represented by a requested element
\item get\+Point\+: Function that returns the n-\/th point for a requested element
\item side\+: Function that returns the side of a requested element, when provided a plane which passes through a point, and a supplied normal (may not be normalized).
\begin{DoxyItemize}
\item Returns 0 if on the plane
\item Returns +1 if on the same side as the normal
\item Returns -\/1 if on the opposite side of the normal
\end{DoxyItemize}
\end{DoxyItemize}

Using this container as a base, additional data may be held in a derived structure for purposes of maintaining state. The idea here is that since your data set is probably large, you do not want to copy it and you probably don\textquotesingle{}t want to store your data in a different type just so you can do geometric searches on it. Instead, you can use the Facade pattern to create the interface described above for your data, and then use the facade as the container for this implementation to perform geometric searches.
